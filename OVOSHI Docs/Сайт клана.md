# Техническое задание на создание сайта клана

## Главная страница

На главной странице сайта должно быть представлено краткое описание клана. Это позволит новым пользователям быстро понять, чем занимается клан и какие у него цели. Запуск сайта должен осуществляться с использованием Docker, что обеспечит удобство в развертывании и управлении приложением.

## Таблица участников

На сайте должна быть реализована таблица участников с возможностью редактирования только для Лидера, который также является администратором сайта. Таблица должна содержать следующие столбцы:

- Никнейм
- UID
- Очки клана
- Карма

Все таблицы на сайте должны поддерживать сортировку по столбцам и иметь фильтры для удобства поиска и анализа данных.

## Профиль пользователя

Каждый пользователь должен иметь возможность создать и редактировать свой профиль, который включает:

- **Никнейм**: задается при регистрации.
- **UID**: пользователь вводит сам, но подтверждается Лидером. Может быть заполнен при регистрации.
- **Почта**: указывается при регистрации.
- **Картинка профиля**: пользователь может загрузить изображение, которое должно быть квадратным. При загрузке должна быть возможность обрезки изображения.
- **Баланс очков**: отображает количество очков, заработанных пользователем.
- **Баланс кармы**: показывает уровень кармы пользователя.
- **Роль**: изначально устанавливается как "гость" до подтверждения UID.

Восстановление пароля должно осуществляться через электронную почту.

## История транзакций

Пользователи должны иметь доступ к истории пополнений и списаний очков и кармы. Для Лидера должна быть доступна общая таблица, отображающая все транзакции участников.

## Кабинет Лидера и заместителей

В кабинете Лидера и его заместителей должен быть дополнительный раздел с задачами:

- **Для заместителей**: задачи по отчетам о проведении конкурсов и исключениях участников.
- **Для Лидера**: задачи по проверке выполнения задач заместителей и выдаче заработной платы.

Эти функции должны помочь в управлении кланом и распределении обязанностей между участниками.

### Безопасность

1. **Управление доступом**: Разграничение прав доступа для различных ролей (гость, участник, заместитель, лидер).

### Коммуникация

1. **Чат**: Чат только в Telegram.
2. **Уведомления**: Система уведомлений о новых сообщениях, изменениях в профиле, обновлениях задач и т.д. Уведомления приходят посредством Telegram-бота.

### Интеграции

1. **Социальные сети**: Возможность интеграции с социальными сетями для регистрации и входа на сайт (Яндекс, ВК, Майл).
2. **API**: Разработка API для интеграции с другими сервисами или приложениями.

### Дизайн и пользовательский интерфейс

1. **Адаптивный дизайн**: Обеспечить корректное отображение сайта на различных устройствах (ПК, планшеты, смартфоны). Особо важно для телефонов.
2. **Тема и кастомизация**: Возможность выбора тем оформления или кастомизации внешнего вида сайта.

### Дополнительные функции

1. **События и календарь**: Раздел для планирования и отображения предстоящих событий, турниров и встреч.
2. **Статистика и аналитика**: Раздел с аналитикой по активности участников, выполнению задач и другим показателям.
3. **Магазин или система наград**: Возможность обмена очков на виртуальные награды или товары. Продавать 60 UC за 300 баллов, 180 UC за 800 баллов, напальчники за 200 баллов. Для админа баллы эквивалентны рублям.

### Вики и блог

1. **Вики**: Раздел для создания и редактирования вики-страниц с использованием разметки Markdown (md).
2. **Блог**: Раздел для публикации статей по PUBG Mobile, ориентированный на привлечение трафика по SEO-запросам, таким как:
   - PUBG Mobile (может быть pubgm)
   - PUBG Mobile вступить в клан
   - PUBG Mobile купить юси
   - PUBG Mobile купить UC
   - PUBG Mobile новости

### Технические требования

1. **Производительность**: Оптимизация сайта для быстрой загрузки и работы.
2. **Резервное копирование**: Автоматическое резервное копирование данных для предотвращения потери информации.
3. **Документация**: Подробная документация по развертыванию, использованию и поддержке сайта.
